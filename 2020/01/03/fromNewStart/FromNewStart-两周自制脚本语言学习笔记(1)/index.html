第一章

一些概念的介绍：

  • 汇编语言与机器语言：
    • 机器语言
      • 本质上是一个很长很长的二进制数字字符串
      • 不容易阅读
      • 也并非人能很好的书写和编码的
    • 汇编语言
      • 用于描述这个巨大的数字字符串
      • 通过汇编程序来讲汇编语言转换成机器语言
  • 解释器与编译器：
    • 都属于语言处理器
    • 解释器
      • 根据程序中的算法执行运算
      • 用于执行程序的软件
      • => 虚拟机
    • 编译器
      • 将某种语言转换成另一种语言
      • 执行的行为叫做编译
      • 源程序不直接转换成机器码的话，则称为源码转换器
      • 比如 babel, 就执行的是编译， js => js

处理过程：

一些理解：

  1. 首先源代码是一大串带缩进、带换行的字符串，也就是我们平时编辑写的那些代码
  2. 之后是分词环节，也就是将源代码这大段的字符串，都拆成一个一个的词法单元，也就是 Token
  • 换行符当做代码行数的分隔符
  • 缩进、空格都当做是 Token 的分隔符
  1. 紧接着进行语法分析
  • 这个环节内，我们将词法单元按某种规则进行组合
  • 生成语法树，也就是 ASTree => 参考链接： AST
  1. 代码生成 & 执行
  • 编译器因为是负责编译代码，而不负责解析执行，所以这一步里，编译器负责的是代码的生成
  • 解释器则需要执行

第二章

第二章主要介绍了我们要设计出的 stone 这门脚本语言的一些语法，以及为什么这样设计这个语法

  1. 不需要静态数据类型
  2. 如果程序中出现了字符串变量相减，则终止程序
  3. 不支持 return，最后一句的计算结果即为返回值
  4. 省去 if 语句和 while 语句的条件表达式两端的括号
  5. 支持注释
  6. 句尾不加分号

第四章

第四章主要讲的就是抽象语法树、BNF, 我们一点一点来看

构造抽象语法树的过程，叫做语法分析 => 这里说的抽象语法树，就是我们说的 AST 了

语法分析的主要任务就是分析单词之间的关系，可以理解成，给 token 归类，检查 token 的每一组，都没有按照规定的语法来写

一般的感觉，有终止符和非终止符之分

个人理解：终止符可以称之为叶节点，非终止符可以称之为树枝节点

抽象语法树仅仅表示的是语法分析的结果，抽象语法树，其实是一种去除了多与信息的抽象树结构

比如 (13 + x) * 2, 其实在词法分析中对 ( ) 这两个括号是解析成 token 的，但是在语法分析时，就已经脱去了

这些括号被抽象成了语法分析书上的前后运算规则，在叶节点和树枝节点上都不会有括号

解析的具体过程

因为域名解析涉及到多级缓存，这里我们不考虑缓存的情况，就只讲一级一级查找的过程

以 www.anho.me 为例，过程如下：

  • 首先回去查找本机 host 文件，如果找到了 host 中配置的 ip 域名映射关系，则直接完成 DNS 解析过程
  • 向上一级查找 Local DNS 服务器，如果找到，则完成解析过程
  • LocalDNS没找到的话，则直接会向跟服务器发起解析请求
  • 跟服务器会给 Local DNS 提供一个 .me 顶级域名服务器的 ip 地址
  • LocalDNS 收到 .me 的 DNS 的 ip 地址后，会向这台 DNS 发起请求
  • 如果这台 .md 的 DNS 服务器无法解析的话，则会返回一个负责解析 anho.me 的 DNS 服务器 ip 地址
  • Local DNS 会继续请求 anho.me 的域名，然后找到 www.anho.me 的 ip，返回给 Local DNS 服务器
  • 找到 www.anho.me 的 ip 后，就可以请求 tcp 连接，然后获取 www.anho.me 的 doc 文档了

DNS 和 CDN 之间的相互作用关系

资源上 CDN 是我们常用的提升网站访问速度的方式，但是之前博主从来没有思考过，为什么重庆的用户，会去获取离他更近的成都的资源，而不去获取离他远些的北京的资源？

博主在被问到的时候，甚至还联想到了资源服务器对这个资源做了 302、307 之类的跳转。。之后仔细想想就发现自己的愚蠢了，都已经到达了那个服务器，还再做重定向。。累不累啊

其实在 DNS 那一层就已经完成了解析，因为对静态资源的请求有了 ip，就直接去下载加载资源了

具体细节就是，会在权限域名服务器那一层，插入一个 CDN 网络中的只能 DNS 负载均衡系统

在这个 DNS 的网络中，会依据请求来源的区域来智能分配到离请求来源区域距离近的服务器，将那个服务器的 ip 来返回，然后再去加载资源

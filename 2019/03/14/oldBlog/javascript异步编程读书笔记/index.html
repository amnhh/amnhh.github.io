延时事件队列

当我们调用到 setTimeout 的时候，会将 setTimeout 当成一个延时事件排入队列，然后继续向下执行，直到在没有任何的代码可以执行的时候，这时候才回去查看延时事件队列，调用当前适合触发的事件的处理器（也就是我们之前 setTimeout 的函数体），例如我们之前，在过了 1000ms 之后，本应该在 500ms 时调用的 setTimeout 就属于当前适合触发的事件

我在 chrome 中做了一个测试，我为一个 logo 绑定了一个事件，但是同时去执行一个 5s 的同步事件, 执行完成后立刻点击 logo

我的点击在 5s 之后才被触发，也就是说所有的同步任务执行完成之后，线程空闲出来了的时候才回去执行我处于延时事件队列中的函数。

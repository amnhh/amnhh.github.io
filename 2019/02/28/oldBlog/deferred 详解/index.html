为这个 deferred 添加 resolve 回调和 reject 回调, 其实在一个 deferred 里面, 会存在这三个回调队列, 分别对应 pending, resolve 和 reject

这里就定义了这几个方法, 其实就是 $.Callbacks().add 方法, 只不过做了对应的回调队列的绑定, 在调用 done 的时候, 会向 resolve 的回调队列里面添加这个函数, 调用 fail 的时候, 会向 reject 的回调队列添加这个函数

而 rejectWith 与 resolveWith 这两个方法, 其实就是 $.Callbacks().fireWith, 会对相应的回调队列里面的函数进行调用, 以此来达到 pending => resolved 的时候调用 done 里面的东西, pending => rejected 的时候调用 fail 里面的东西

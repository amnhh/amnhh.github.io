一年前面试的时候，面试官问过一道 let、 var 有什么区别？突然一片空白，想不起来到底是啥区别了。。好在面试官比较好，放了我一马

这次就彻彻底底的来总结一下子，尽量用通俗一点的话来说

var

首先我们来看 var。

var 没有块级作用域，所以我们之前这样的代码的表现可能不像是我们预期的那样：

let

我们再来看一下 let。

let 也是用来声明变量，我们可以把它看成一个更高级的 var, 声明变量的同时，添加了一些规则在其中

它带有块级作用域，且不会进行变量提升，且在执行到 let 声明语句之前，都处在临时性死区(TDZ) 中，let 后的标识符在 TDZ 中不可读写

这里我们把 i 包裹在一个大括号内，则 i 的存在的作用域就是这个大括号，所以我们在外部试图访问 i 的时候，会发现 i 是 undeclared 状态

这里我们在 let 声明之前，尝试去读取 let 后跟随的那个标识符的值，也一样会报错，因为此时 i 处在 TDZ 中，不可读不可写

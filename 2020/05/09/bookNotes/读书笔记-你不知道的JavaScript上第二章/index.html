对词法作用域的理解？

  • 就是定义在词法阶段的作用域
  • 写代码的时候决定的
  • 词法分析器在执行的时候，会保证这一作用域的嵌套关系保持不变

唯一性

没有任何函数可以同时出现与两个直接父级元素中

理解：同时出现在两个的时候，你咋写？我感觉我是不会写的….

作用域里查找标识符？

  • 以标识符的名字为查找的根本
  • 查找的顺序为由内到外
  • 查找过程会在两种情况下停止
    • 找到 第一个 匹配的标识符 => 这也就造成了内部同名标识符覆盖外部同名标识符的现象
    • 在全局环境中也没找到…
  • 全局变量有个好处：
    • case 是在我们查找的时候，标识符 example 会遮蔽到外部的同名属性，所以在内部是根本拿不到外部的 example 标识符的
    • 但是这时候如果 example 在全局中，我们可以通过 window.example 拿到
    • 只有全局变量才有这个优势223333
  • 不论何时被调用，词法作用域都只由函数被声明时所处的位置决定

欺骗词法

  • eval
  • with

因为 js 会在词法分析的时候进行大量的性能优化，而 eval 和 with 使用中，并不知道 with 的，或者 eval 的是什么，所以最简单的做法就是完全不做任何优化2333

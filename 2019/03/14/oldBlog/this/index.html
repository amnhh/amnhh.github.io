以下都是我本人的理解, 有不对的地方还请指出, 谢谢

首先, 函数调用的时候才会涉及到 this 的各种问题, 不在函数调用里面, 都会指向全局对象, browser 里的 window 和 node 环境下的 global

我也做了一些 demo 去测试, 最终还是得到了一个老生常谈的结论, 在不 call 或者 apply 的情况下, 谁调用, this 就指向谁

本文的 demo 例子都是在 chrome 环境下测试的, 控制台还是方便…

有了 prototype, 再来个 new 呢？

可以思考一下, 这里的 this 指向哪里呢? 我们在 new 的过程中, 其实做了这几件事

  • 创建一个全新的对象
  • 这个新对象会被执行 [[Prototype]] 连接
  • 构造函数中的 this 会指向这个新对象
  • 如果函数没有显式的返回其他对象, 那么 new 表达式中的函数调用会自动返回这个新对象

这四件事里面的第三件就是这里的答案了, 在 new 的过程中, 会将构造函数里面代码执行的时候的 this 全部绑定到这个新的实例上, 也就是说, 在 new 的过程中, 函数中 this 指向实例

所以上面那个例子返回的就是 true 咯

既然提到了 new, 就把这四条全部来解释一下

创建一个全新的对象

这句不用过多解释, 相当于 var obj = {}; 这样， 创建一个 Object 的实例

这个新对象会被执行 [[Prototype]] 连接

首先说这个 [[Prototype]], 我们都知道原型链这个东西, 原型链的寻找过程其实不是 xxx.prototype 指向的那个东西依次向上找, 而是一个内部的指针 [[Prototype]] 的依次向上寻找, 也就是各大浏览器里面实现的 __proto__ 这个东西

再返回来说这句话, 新对象会被执行 [[Prototype]] 连接, 其实做的事就是

就只是把内部的 [[Prototype]] 指针指向构造函数的 prototype

新对象绑定到函数调用的 this

这句话其实也就是上面有解释过得, 在 new 后面跟的那个构造函数, 在这条语句执行之前, 内部的 this 指向全部都被指定为新生成的这个 obj

如果函数没有返回其他对象, 那么 new 表达式中的函数调用会自动返回这个新对象

这句话的意思, 就是说, 我在构造函数的最后 return {obj : 1} 出来, 最后生成的不会是那个实例, 而会是 {obj : 1}

需要注意的一点就是, 这里所说的没有返回其他对象, 必须是对象, 加入你是 return 1, 还是会生成实例而不是那个 1

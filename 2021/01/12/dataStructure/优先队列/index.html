优先队列

2021-01-12

1. 定义

优先队列是一种抽象的数据类型，其中每个元素都拥有各自的优先级，优先级最高的元素最先得到服务（来自维基百科）
可以理解为，优先队列首先是一种队列，支持队列的各种操作，其次，队列中需要对元素的优先性，有一个先后关系。

2. 适用场景举例

这里举两个例子来辅助理解：

例 1：
打印机的打印的先后，依赖于哪个任务先到达打印机，但是任务有加急的区分，所以此时需要把加急的任务放在前面

例 2：
再例如双十一抢购商品，一般不会实时操作库表，而是收集请求后异步处理，在一整个订单队列中，高级 vip 需要优先处理，也就是高级 vip 抢购商品拥有优先的权利，所以此时的队列需要维护成一个优先队列而并非一个普通队列

3. 实现

一般我们会使用二叉堆来实现优先队列，使用完全二叉树来实现二叉堆。
而完全二叉树，则是从上到下，从左到右依次写入元素，所以我们可以使用数组作为底层数据结构使用，例如：
(图片来自《算法第四版》)

使用这种存储方式，我们可以发现一些特点和性质：

  • 0 位置不存元素，树的根节点存储在 1 位置上
  • 若一个节点的索引为 i，则其父节点索引为 i / 2
  • 若一个节点的索引为 i，则其左右孩子节点索引分别为 2 _ i 和 2 _ i + 1
  • 一个节点的优先级总高于其左右孩子节点的优先级

首先我们可以认为插入前，整个二叉堆都满足「每个节点的优先级均大于其左右孩子节点的优先级」。虽然并不是广义上的有序，但是却基于优先级存在一定的顺序，所以我们不能够随便找个坑位就插入。

下面说回我们之前略过的 heapify 方法。这个方法用于带数据域的构造方法中，会将杂乱的集合进行堆化，使之满足二叉堆的性质。
我们后续将输入设置为 N。
其实可以有两条主要的思路：

  1. 将 N 个元素从左到右依次执行 add -> siftUp 操作 => N 个元素每次插入为 logN 复杂度，整体堆化完成需要 NlogN
  2. 将 N 个元素，从右至左，依次执行 siftDown 操作 => N 个元素，共有 N/2 个需要进行下沉操作，每次下沉操作需要 logN ，所以总体为 N / 2 * logN
